# ルーター・ファームウェア脆弱性シミュレーター

社会情報学実習 #1（サイバーセキュリティ）の発展課題用に作った、**完全オフラインの教育用Webアプリ**です。
年代の異なるルーターのファームウェアに、歴史上有名な攻撃手法を当てて「どの時代に何が効くか」を安全に観察できます。

> 授業の上級課題「自宅ブロードバンドルーターのファームウェアをアップデートしてみよう」の **裏返し**＝
> *更新せず古いままだと何が起きるのか* を可視化します。

---

## なぜシミュレーターなのか（このアプリの動機）

実機で「わざと脆弱性を作って過去の手法を試す」のは、

- 改造／古いファームは **技適（電波法の技術基準適合）** 上グレーになりがち
- 攻撃トラフィックが **外部に漏れる** と重大な問題

というリスクがあります。
このアプリは **ブラウザの中だけ** で完結する模擬実験なので、これらの懸念が原理的に発生しません。

- 実ネットワーク通信を **一切しない**（`fetch`・`WebSocket` 不使用）
- 表示されるIP・認証情報・コンソール出力・脆弱性はすべて **架空の模擬データ**
- そのまま実機で動く攻撃コードは含まない（手法の「仕組みと歴史」を学ぶための再現）

---

## 使い方

1. `index.html` を **ダブルクリック** してブラウザで開くだけ（サーバー不要・インストール不要）。
2. 上部タブの構成：
   - **シミュレーター** … 年代を選び → 攻撃手法を選び → 「▶ 実行」でコンソールに結果が流れる
   - **年代 × 攻撃マトリクス** … 全組み合わせの成否を一覧。セルを押すとその場で再現
   - **解説・年表** … ハッカー文化史〜ルーター史の年表と、各手法の仕組み・実例・対策
   - **このアプリについて** … 目的・安全性・倫理上の注意

> ローカルで開いて文字化けする場合は、簡易サーバー経由が確実です。
> `python -m http.server` をこのフォルダで実行し、`http://localhost:8000/` を開いてください。

## 収録している年代（ファームウェア）

| 年代 | バージョン | 特徴 |
|------|-----------|------|
| 2004 | v1.0 黎明期 | HTTP平文・admin/admin・Telnet開放・WEP |
| 2008 | v2.2 WPS登場期 | WPS有効・初期パスワードのまま |
| 2012 | v3.5 バックドア時代 | 隠しバックドア・組込みサーバ欠陥・WPS総当たり |
| 2016 | v4.1 Mirai襲来 | 初期パスワードの甘さが残る過渡期 |
| 2020 | v5.3 自動更新時代 | 端末固有パスワード・Telnet廃止・署名付き更新 |
| 2024 | v6.0 現代 | セキュアブート・WPA3・HTTPSのみ |

## 収録している攻撃手法

初期パスワード / Telnet・バックドア / WPS PIN総当たり(Reaver) / コマンドインジェクション /
CSRFによるDNS改ざん / 認証バイパス / TCP32764バックドア / Misfortune Cookie /
UPnP悪用 / Mirai型ボットネット / Wi-Fi暗号解読 / 改造ファーム書き込み

## ファイル構成と拡張方法

```
社会情報学実習アプリケーション/
├── index.html          画面の骨組み
├── css/style.css       見た目
└── js/
    ├── firmwares.js    年代（ファームウェア）の定義 … flags を足し引きするだけで挙動が変わる
    ├── attacks.js      攻撃手法の定義 … evaluate(fw) が成否とコンソール出力を返す
    └── app.js          UI構築・実行エンジン・マトリクス・解説の生成
```

- **年代を足す**：`firmwares.js` の配列に1要素追加（`flags` を設定）。マトリクスに自動反映。
- **手法を足す**：`attacks.js` の配列に1要素追加（`evaluate(fw)` を実装）。同じく自動反映。
- 新しいセキュリティ性質を表現したいときは、`flags` に新キーを足し、`FLAG_DESCRIPTORS`（表示用）と各 `evaluate` で参照する。

## ⚠️ 倫理・法律上の注意

ここで学ぶ手法を、**自分が管理していない機器やネットワークに対して使うことは違法**です（不正アクセス禁止法など）。
学習・研究は必ず **自分の所有物・許可を得た環境・本物に触れない模擬環境** で行ってください。
このアプリ自体は、その「本物に触れない模擬環境」を提供することを目的としています。
